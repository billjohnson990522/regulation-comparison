## 二、数据结构

### 输入数据格式（JSON）
```json
{
  "level": 1,                    // 层级：1=章, 2=条, 3=款
  "number": "第一章",            // 编号
  "title": "总则",               // 标题
  "content": "...",              // 内容
  "children": [                  // 子项列表
    {
      "level": 2,
      "number": "第一条",
      "title": "为了规范...",
      "content": "",
      "children": [...]
    }
  ]

```

### 层级说明
| 层级 | level值 | 编号格式 | 说明 |
|------|---------|---------|------|
| 一级标题 | 1 | 第一章、第二章... | 章/章节 |
| 二级标题 | 2 | 第一条、第二条... | 条/条款 |
| 三级标题 | 3 | （一）、（二）... | 款/款项 |
| 四级标题 | 4 | 1.、2.、1、2、... | 目/项目 |

## 三、核心匹配算法

### 3.1 三轮匹配策略

```
┌─────────────────────────────────────────────────────────────┐
│                     三轮匹配算法                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一轮：编号精确匹配                                        │
│  ├── 遍历老法规所有项目                                      │
│  ├── 按number字段（如"第一条"）查找新法规中相同编号          │
│  ├── 找到 → 标记为已匹配，进入内容对比                       │
│  └── 未找到 → 进入第二轮                                    │
│                                                             │
│  第二轮：相似度匹配                                          │
│  ├── 对未匹配的老法规项目，计算与新法规未匹配项目的相似度    │
│  ├── 使用difflib.SequenceMatcher计算title相似度             │
│  ├── 相似度 ≥ 50% → 判定为修改/移动，标记已匹配              │
│  └── 相似度 < 50% → 进入第三轮                              │
│                                                             │
│  第三轮：新增/删除判定                                       │
│  ├── 老法规有、新法规无 → 判定为删除                         │
│  └── 新法规有、老法规无 → 判定为新增                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 为什么需要相似度匹配？

**问题场景**：假设老法规有第1-10条，新法规删除了第5条

```
老法规：第1条、第2条、第3条、第4条、第5条、第6条...
新法规：第1条、第2条、第3条、第4条、第6条、第7条...
```

如果只用编号匹配：
- 第1-4条：正常匹配 ✓
- 第5条：老法规有，新法规无 → 删除 ✓
- 第6条：编号相同但内容不同（老第6条实际是新第5条的内容）→ 误判！

**解决方案**：使用相似度匹配
- 老第6条 vs 新第6条 → title相似度低 → 继续查找
- 老第6条 vs 新第5条 → title相似度高 → 判定为移动/修改

### 3.3 相似度计算方法

```python
def _calculate_similarity(self, text1: str, text2: str) -> float:
    """
    使用difflib.SequenceMatcher计算文本相似度
    返回值范围：0.0 ~ 1.0
    """
    matcher = difflib.SequenceMatcher(None, text1, text2)
    return matcher.ratio()
```

## 十、常见问题处理

### Q1: 如何处理编号位移？

当新增或删除条款导致后续条款编号变化时：
1. 第一轮编号匹配会失败
2. 第二轮相似度匹配会找到实际对应关系
3. 标记为"修改"并记录编号变化

### Q2: 如何判断是新增还是修改？

- 新增：老法规中不存在任何相似内容（相似度 < 50%）
- 修改：老法规中存在相似内容（相似度 ≥ 50%）

### Q3: 三级标题为什么要对比content？

三级标题通常是具体款项，可能包含详细内容。仅对比title可能遗漏内容变化，因此需要同时对比content。

### Q4: AI总结失败怎么办？

程序会捕获异常并继续执行，失败的变更项会标记"[AI总结失败]"，不影响其他功能。
